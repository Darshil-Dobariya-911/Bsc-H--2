1. Introduction to File Handling in C

File handling in C allows a program to persist data beyond program execution by reading from and writing to files stored on disk. The C standard library (<stdio.h>) provides a rich set of functions that abstract low-level OS system calls into portable, easy-to-use routines.

All file I/O in C revolves around the FILE structure — an opaque type defined in <stdio.h> that contains all the metadata the C runtime needs to manage an open file (buffer, position indicator, error flags, etc.). Programs never access this structure directly; they always use a pointer to it: FILE *fp;.

Key header required Every file-handling program must include #include <stdio.h>. This header declares the FILE type, EOF macro, and all I/O functions discussed in this document.

2. File Operation Lifecycle

Every file interaction in C follows a strict three-step lifecycle:

①  Open the file — fopen()
②  Read / Write data — fgetc, fgets, fscanf, fprintf, fputc …
③  Close the file — fclose()
⚠ Always check for NULL after fopen() If fopen() fails (file missing, permission denied, disk full) it returns NULL. Attempting to read/write through a NULL pointer causes undefined behaviour — always validate first.

3. fopen() – Opening a File

Open

Syntax

FILE *fopen(const char *filename, const char *mode);

Parameters

ParameterTypeDescription
filenameconst char *Path/name of the file to open (e.g. "output.txt")
modeconst char *String specifying how the file should be opened (see §7)

Return Value

Returns a FILE * pointer on success, or NULL on failure.

Example (from a1.c)

FILE *fp; fp = fopen("output.txt", "w"); // open for writing if (fp == NULL) { printf("File cannot be opened!\n"); return 1; }

4. fclose() – Closing a File

Close

Syntax

int fclose(FILE *fp);

Description

Flushes any pending writes in the buffer to disk, releases all resources associated with the stream and invalidates the pointer. Returns 0 on success, EOF on error.

✅ Best practice Always call fclose() for every successfully opened file. Forgetting to close a file may result in data loss (unflushed buffers) and resource leaks (each process has a limited number of open file descriptors).
fclose(fp); // seen in every program: a1.c, a2.c, a3.c, a4.c, b1.c, b2.c, b3.c

5. Writing Functions

5.1 fprintf()  Write

fprintf() is the file-writing counterpart of printf(). It writes a formatted string to a file stream using the same format specifiers as printf.

Syntax

int fprintf(FILE *fp, const char *format, ...);
ParameterDescription
fpPointer to the open FILE (must be opened in write/append mode)
formatFormat string with optional conversion specifiers (%s, %d, %f, …)
...Variable arguments matching the format specifiers

Return: Number of characters written, or a negative value on error.

Usage in Lab Programs

// a1.c – write a string fprintf(fp, "%s", str); // a3.c – append a line fprintf(fp, "%s\n", str); // b3.c – write structured record fprintf(fp, "%s %d %.2f\n", s.name, s.id, s.grade);
💡 fprintf vs printf The only difference between fprintf and printf is the first argument: printf always writes to stdout, while fprintf accepts any FILE pointer — including stdout and stderr.

5.2 fputc()  Write

Writes a single character to the specified file stream. It is the simplest write function and is ideal when copying files character by character (as in b1.c).

int fputc(int ch, FILE *fp);
ParameterDescription
chCharacter to write (passed as int, internally cast to unsigned char)
fpPointer to the target file stream

Return: The written character on success, EOF on failure.

// b1.c – copy file1 into merged.txt one character at a time while ((ch = fgetc(f1)) != EOF) { fputc(ch, f3); // write each character to output file }

5.3 fputs()  Write

Writes a null-terminated string (without the trailing \0) to the file. It does not automatically append a newline.

int fputs(const char *str, FILE *fp);

Return: Non-negative value on success, EOF on error.

Note None of the seven lab programs use fputs() directly, but it is the character-string counterpart of fgets() and is commonly used with fgets()-based reading loops.

6. Reading Functions

6.1 fgetc()  Read

Reads and returns the next single character from the file stream, advancing the file position indicator by one byte. Returns EOF (typically –1) when the end of file is reached or an error occurs.

int fgetc(FILE *fp);
Return valueMeaning
Character (0–255)Successfully read character
EOF (−1)End of file reached, or I/O error

Usage in Lab Programs

// a2.c – print every character of input.txt while ((ch = fgetc(fp)) != EOF) { printf("%c", ch); } // b1.c – copy characters from file1 & file2 into merged.txt while ((ch = fgetc(f1)) != EOF) { fputc(ch, f3); }
⚠ Store result in int, not char fgetc() returns an int so it can represent both all possible byte values (0–255) and the special EOF (–1) sentinel. Storing the return value in a char variable can hide the EOF signal if char is unsigned on the target platform.

6.2 fgets()  Read

Reads at most n-1 characters from the file into a buffer, stopping early if a newline or EOF is encountered. The newline, if read, is stored in the buffer. A null terminator is always appended.

char *fgets(char *buffer, int n, FILE *fp);
ParameterDescription
bufferDestination character array
nMaximum characters to read (including null terminator)
fpSource file stream

Return: buffer on success, NULL at EOF or error.

Usage in Lab Programs

// a4.c – read file line by line to search for a word while (fgets(line, sizeof(line), fp)) { lineNumber++; if (strstr(line, word) != NULL) { printf("Word found at line %d\n", lineNumber); } }
✅ fgets is safer than gets() fgets() accepts a size argument and will never overflow the buffer — unlike the now-deprecated gets() function used in a1.c/a3.c for stdin input, which has no bounds check and is a classic security vulnerability (buffer overflow).

6.3 fscanf()  Read

The file-reading counterpart of scanf(). Reads formatted input from a file, parsing tokens according to format specifiers, and skipping leading whitespace (spaces, tabs, newlines) by default.

int fscanf(FILE *fp, const char *format, ...);

Return: Number of items successfully matched and assigned. Returns EOF when end-of-file is reached before any match.

Usage in Lab Programs

// b2.c – count words in a file while (fscanf(fp, "%99s", word) == 1) { count++; } // Return values: // 1 → one word successfully read ✅ // EOF → end of file reached ❌ // 0 → input did not match format ❌
💡 Why compare to 1 and not EOF? Comparing fscanf(...) == 1 is more robust than != EOF because it also catches format-mismatch failures (return value 0), whereas checking only for EOF would loop forever on a malformed file.

8. Program-by-Program Analysis

a1.c — Write a User String to a File Functions: fopen · fprintf · fclose

Purpose: Accepts a string from the user and writes it to output.txt.

// Full source — a1.c #include <stdio.h> int main() { FILE *fp; char str[100]; printf("Enter a string: "); gets(str); // reads from stdin (use fgets for safety) fp = fopen("output.txt", "w"); if (fp == NULL) { printf("File cannot be opened!"); return 1; } fprintf(fp, "%s", str); // write string to file fclose(fp); printf("Data written to file successfully!"); return 0; }

⚠ Caution gets() does not perform bounds checking and is removed in C11. Replace with fgets(str, sizeof(str), stdin).

a3.c — Append Data to an Existing File Functions: fopen("a") · fprintf · fclose

Purpose: Opens log.txt in append mode and adds a new line without overwriting existing content.

fp = fopen("log.txt", "a"); // "a" preserves existing content fprintf(fp, "%s\n", str); // \n ensures each entry is on its own line fclose(fp);
✅ Difference between "w" and "a" Mode "w" destroys existing file contents. Mode "a" always positions the write cursor at the end of the file, making it perfect for log files that must grow over multiple program runs.
a4.c — Search for a Word in a File Functions: fopen · fgets · strstr · fclose

Purpose: Reads text.txt line by line using fgets(); uses strstr() from <string.h> to search for a keyword and report its line number.

while (fgets(line, sizeof(line), fp)) { lineNumber++; if (strstr(line, word) != NULL) { printf("Word found at line %d\n", lineNumber); found = 1; } } // strstr(haystack, needle) – returns pointer to first match, or NULL

Key insight: fgets() preserves newlines, so each call maps exactly to one line of the file — ideal for line-number tracking.

b1.c — Merge Two Files into One Functions: fopen(×3) · fgetc · fputc · fclose(×3)

Purpose: Opens file1.txt and file2.txt for reading, and merged.txt for writing. Copies each source file character-by-character into the destination.

FILE *f1, *f2, *f3; f1 = fopen("file1.txt", "r"); f2 = fopen("file2.txt", "r"); f3 = fopen("merged.txt", "w"); while ((ch = fgetc(f1)) != EOF) fputc(ch, f3); while ((ch = fgetc(f2)) != EOF) fputc(ch, f3); fclose(f1); fclose(f2); fclose(f3);

This demonstrates that multiple FILE pointers can be open simultaneously. The fgetc→fputc pipeline is the simplest portable file-copy mechanism in C.

b2.c — Count Words in a File Functions: fopen · fscanf · fclose

Purpose: Counts the total number of whitespace-delimited words in file.txt using fscanf().

while (fscanf(fp, "%99s", word) == 1) { count++; } printf("Total words = %d", count);
fscanf ReturnMeaningLoop action
1Word successfully readContinue, increment count
EOFEnd of fileExit loop
0Format mismatchExit loop (safe)
Format width specifier %99s limits the read to 99 characters (+ null terminator = 100), preventing buffer overflow when the word array is char word[100].
b3.c — Store Student Records to File Functions: fopen · scanf · fprintf · fclose

Purpose: Accepts multiple student records (Name, ID, Grade) from the user and stores them in students.txt using fprintf().

struct Student { char name[50]; int id; float grade; }; fp = fopen("students.txt", "w"); for (i = 0; i < n; i++) { scanf("%s %d %f", s.name, &s.id, &s.grade); fprintf(fp, "%s %d %.2f\n", s.name, s.id, s.grade); }

This program demonstrates writing structured (record-based) data in text format. Each record is written as one line with space-separated fields, making it easy to read back with fscanf() or parse in other languages.

%.2f format specifier %.2f formats the float to exactly two decimal places, which is important for consistent file formatting when reading records back.

10. Common Errors & Best Practices

10.1 Always Validate fopen()

// ❌ Wrong – no NULL check fp = fopen("data.txt", "r"); fgetc(fp); // undefined behaviour if file missing // ✅ Correct fp = fopen("data.txt", "r"); if (fp == NULL) { perror("fopen"); return 1; }

10.2 Replace gets() with fgets()

// ❌ Dangerous (buffer overflow) gets(str); // ✅ Safe fgets(str, sizeof(str), stdin);

10.3 Store fgetc() Result in int

// ❌ Wrong – may never equal EOF on unsigned-char platforms char ch; while ((ch = fgetc(fp)) != EOF) ... // ✅ Correct int ch; while ((ch = fgetc(fp)) != EOF) ...

10.4 Use Width Specifiers with fscanf

// ❌ Risky – no width limit fscanf(fp, "%s", word); // ✅ Safe – limits input to buffer size - 1 fscanf(fp, "%99s", word); // for char word[100]

10.5 Always Close Files

// After all operations are done: fclose(fp); // flushes internal buffer → data guaranteed on disk

Key Takeaways
  • Always include <stdio.h> for file I/O functions.
  • Every file must be opened with fopen() and the returned pointer checked for NULL.
  • Use "r" to read, "w" to write (truncates), "a" to append.
  • Use fgetc/fputc for character-level, fgets for line-level, fscanf/fprintf for formatted I/O.
  • Always close files with fclose() to flush buffers and free resources.
  • Prefer fgets() over gets() to avoid buffer overflows.
