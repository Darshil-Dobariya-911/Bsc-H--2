1. What is Dynamic Memory Allocation?

In C, memory can be managed in two ways: statically (size fixed at compile time) or dynamically (size decided at run time). Dynamic Memory Allocation (DMA) lets a program request memory from the heap while it is running, use it, and then release it back.

Key Insight When you write int arr[10]; the compiler reserves exactly 40 bytes on the stack before the program even starts. With DMA you can ask: "Give me space for exactly n integers" — where n is typed by the user at run-time.

Heap vs Stack The stack is managed automatically — memory is freed when a function returns. The heap is manual — you allocate with malloc() and must free with free(). Forgetting to free causes a memory leak.

4. How malloc() Works – Step by Step

  1. 1 You call malloc(n * sizeof(int)).
  2. 2 The C runtime asks the OS for a contiguous block of n × 4 bytes on the heap.
  3. 3 malloc returns a void* pointer to the start of that block, or NULL on failure.
  4. 4 You cast the void* to the desired type (e.g., int*).
  5. 5 You use the memory like a normal array.
  6. 6 You call free(ptr) when done — the block returns to the heap pool.
Why cast? malloc always returns void* — a generic pointer. Casting (int *) tells the compiler "treat these bytes as integers." In C this cast is optional but considered good style.
Program A1
Dynamic 1-D Array using malloc()

Objective

Allocate a 1-D integer array of user-defined size at run-time, store elements, display them, and free the memory.

a1.c#include <stdio.h>
#include <stdlib.h>

int main() {
    int n, i;
    int *arr;

    printf("Enter size of array: ");
    scanf("%d", &n);

    arr = (int *)malloc(n * sizeof(int)); // allocate n integers

    if (arr == NULL) {               // always check!
        printf("Memory allocation failed!\n");
        return 1;
    }

    printf("Enter %d elements:\n", n);
    for(i = 0; i < n; i++)
        scanf("%d", &arr[i]);

    printf("Array elements are:\n");
    for(i = 0; i < n; i++)
        printf("%d ", arr[i]);

    free(arr);     // release memory back to heap
    return 0;
}

Line-by-Line Explanation

LineExplanation
int *arr;Declares a pointer. No memory allocated yet — just a variable to hold an address.
malloc(n * sizeof(int))Requests n × 4 bytes from the heap (sizeof int = 4 bytes on most systems).
(int *) castConverts the generic void* return into an int* so we can use array indexing.
if (arr == NULL)Critical safety check. If allocation fails (e.g., out of memory), malloc returns NULL.
arr[i]The pointer is used exactly like an array. Internally, arr[i] is *(arr + i).
free(arr)Returns the allocated block to the heap. Without this, we have a memory leak.
Sample Run Enter size of array: 4  →  Elements: 10 20 30 40  →  Output: 10 20 30 40
Program A2
Dynamic 2-D Matrix – Sum of Elements

Objective

Allocate a 2-D matrix of m × n integers dynamically using a pointer-to-pointer, read elements, compute their sum, and properly free everything.

a2.c#include <stdio.h>
#include <stdlib.h>

int main() {
    int m, n, i, j, sum = 0;
    int **matrix;       // pointer to pointer (2-D)

    printf("Enter rows and columns: ");
    scanf("%d %d", &m, &n);

    // Step 1: allocate array of m row-pointers
    matrix = (int **)malloc(m * sizeof(int *));

    // Step 2: allocate each row of n columns
    for(i = 0; i < m; i++)
        matrix[i] = (int *)malloc(n * sizeof(int));

    printf("Enter matrix elements:\n");
    for(i = 0; i < m; i++)
        for(j = 0; j < n; j++) {
            scanf("%d", &matrix[i][j]);
            sum += matrix[i][j];
        }

    printf("Sum of all elements = %d\n", sum);

    // Free in reverse order: rows first, then the pointer array
    for(i = 0; i < m; i++)
        free(matrix[i]);
    free(matrix);

    return 0;
}

2-D Memory Model

A 2-D matrix is NOT one big block. It is an array of pointers, each pointing to its own row:

matrix  →  [ ptr0 | ptr1 | ptr2 ]        ← 1st malloc (array of pointers)
              |       |       |
              ↓       ↓       ↓
           [c0,c1] [c0,c1] [c0,c1]       ← each row: separate malloc

Key Points

ConceptDetail
int **matrixA pointer to a pointer — stores the address of the row-pointer array.
First mallocm * sizeof(int*) — space for m pointers (one per row).
Second malloc (loop)Each matrix[i] gets its own block of n integers.
Freeing orderMust free each matrix[i] before freeing matrix. Doing it in reverse avoids dangling pointer issues.
Common Mistake Calling free(matrix) first loses the row pointers, causing a memory leak of every individual row. Always free child allocations before parent!
Program B1
Dynamic String – Reverse using malloc()

Objective

Dynamically allocate a character buffer based on user-specified length, read a string, reverse it in-place without extra memory, and free the buffer.

b1.c#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main() {
    int n, i;
    char *str;

    printf("Enter length of string: ");
    scanf("%d", &n);

    // +1 for the null terminator '\0'
    str = (char *)malloc((n + 1) * sizeof(char));

    if (str == NULL) {
        printf("Memory allocation failed!\n");
        return 1;
    }

    printf("Enter string: ");
    scanf(" %[^\n]", str);   // reads full line including spaces

    printf("Original string: %s\n", str);

    // In-place reversal using two-pointer swap
    for(i = 0; i < n / 2; i++) {
        char temp   = str[i];
        str[i]       = str[n - i - 1];
        str[n-i-1]  = temp;
    }

    printf("Reversed string: %s\n", str);

    free(str);
    return 0;
}

Critical Concepts

DetailWhy It Matters
n + 1 bytesEvery C string ends with a null character '\0'. Allocating only n bytes would cause a buffer overrun.
sizeof(char) == 1Always 1 byte. Writing it explicitly makes the code self-documenting.
%[^\n] formatReads the entire line including spaces, unlike %s which stops at whitespace.
In-place reversalUses only one char temp – no second buffer. Iterates only half the array (n/2 swaps).
Sample Run Length: 5  |  String: "Hello"  →  Reversed: "olleH"

Complete Example – Student Marks

Allocate an array of student marks at run-time, compute the average, then release the memory.

malloc_demo.c#include <stdio.h>
#include <stdlib.h>

int main() {
    int   n, i;
    float sum = 0, avg;
    int  *marks;

    printf("Enter number of students: ");
    scanf("%d", &n);

    /* malloc returns void* — cast to int* */
    marks = (int *)malloc(n * sizeof(int));

    /* ALWAYS check before using the pointer */
    if (marks == NULL) {
        printf("Memory allocation failed!\n");
        return 1;
    }

    printf("Enter marks for %d students:\n", n);
    for (i = 0; i < n; i++) {
        scanf("%d", &marks[i]);
        sum += marks[i];
    }

    avg = sum / n;
    printf("Average marks = %.2f\n", avg);

    free(marks);   // release heap memory
    marks = NULL; // avoid dangling pointer
    return 0;
}

How malloc() Works – Internals

  1. 1 malloc(n * sizeof(int)) computes the total bytes needed.
  2. 2 The C runtime scans its internal free-list on the heap for a large enough block.
  3. 3 If found, it marks that block as "used" and returns a pointer to it.
  4. 4 If not found, it requests more memory from the OS via sbrk() / mmap().
  5. 5 The returned void* is cast to the required type (int*, char*, etc.).
  6. 6 Contents are uninitialised — contain whatever was previously in that memory.
AspectDetail
Header required#include <stdlib.h>
Return typevoid* — must be cast to the correct pointer type
On failureReturns NULL — always check before dereferencing
Initialisation❌ No — memory contains garbage (random old values)
LifetimePersists until free() is called — outlives function scope
Best forWhen you will fill every element immediately (no need for zero-init)
⚠ malloc does NOT initialise memory Reading from a freshly malloc'd block without writing first gives unpredictable (garbage) values. If you need zeroed memory, use calloc() instead.

Zero-Initialised Array using calloc()

Objective

Use calloc() to allocate an integer array that is automatically initialised to zero, demonstrating the key difference from malloc().

calloc_demo.c#include <stdio.h>
#include <stdlib.h>

int main() {
    int n, i;
    int *arr;

    printf("Enter number of elements: ");
    scanf("%d", &n);

    /* calloc(count, size_per_element) — initialises all bytes to 0 */
    arr = (int *)calloc(n, sizeof(int));

    if (arr == NULL) {
        printf("Memory allocation failed!\n");
        return 1;
    }

    /* All elements are already 0 — no explicit initialisation needed */
    printf("Default values after calloc:\n");
    for(i = 0; i < n; i++)
        printf("%d ", arr[i]);   // prints: 0 0 0 0 ...

    printf("\nEnter %d new values:\n", n);
    for(i = 0; i < n; i++)
        scanf("%d", &arr[i]);

    printf("Updated array:\n");
    for(i = 0; i < n; i++)
        printf("%d ", arr[i]);

    free(arr);
    return 0;
}

calloc vs malloc – Key Differences

Aspectmalloc(size)calloc(n, size)
ArgumentsTotal bytes as one valueCount & size separately (safer)
Initialisation❌ Garbage / unpredictable values✅ All bytes set to 0
SpeedSlightly faster (no zeroing)Slightly slower (fills with zeros)

Program C2
Resizing Memory at Runtime using realloc()

Objective

Start with a small array, fill it, then expand its size using realloc() to add more elements — without losing original data.

realloc_demo.c#include <stdio.h>
#include <stdlib.h>

int main() {
    int i;

    /* Step 1: allocate space for 3 integers */
    int *arr = (int *)malloc(3 * sizeof(int));
    if (!arr) { printf("malloc failed\n"); return 1; }

    arr[0] = 10;  arr[1] = 20;  arr[2] = 30;
    printf("Original (3 elements): ");
    for(i=0; i<3; i++) printf("%d ", arr[i]);

    /* Step 2: grow the block to hold 6 integers */
    int *temp = (int *)realloc(arr, 6 * sizeof(int));
    if (!temp) {
        printf("\nrealloc failed\n");
        free(arr);   // free original if realloc fails
        return 1;
    }
    arr = temp;   // safe to reassign now

    arr[3] = 40;  arr[4] = 50;  arr[5] = 60;
    printf("\nExpanded (6 elements): ");
    for(i=0; i<6; i++) printf("%d ", arr[i]);

    /* Step 3: shrink back to 2 elements */
    arr = (int *)realloc(arr, 2 * sizeof(int));
    printf("\nShrunk  (2 elements): ");
    for(i=0; i<2; i++) printf("%d ", arr[i]);

    free(arr);
    return 0;
}

How realloc() Works Internally

  1. 1 realloc(ptr, newSize) is called with an existing pointer and desired new size.
  2. 2 If the heap block can be extended in-place, it does so and returns the same address.
  3. 3 If not, it allocates a new block elsewhere, copies old data, frees the old block, and returns the new pointer.
  4. 4 If newSize == 0, it behaves like free(ptr).
  5. 5 If ptr == NULL, it behaves like malloc(newSize).
ScenarioBehaviour
Expand (grow)Old data preserved; new bytes contain garbage
Shrink (reduce)Only first newSize bytes kept; rest returned to heap
FailureReturns NULL; original block is NOT freed
realloc(NULL, n)Equivalent to malloc(n)
realloc(ptr, 0)Equivalent to free(ptr)
⚠ Critical Safety Pattern Never write arr = realloc(arr, newSize) directly! If realloc fails it returns NULL and you lose the original pointer — a memory leak. Always store the result in a temporary pointer first, check it, then assign to arr.
Sample Run Original: 10 20 30  →  Expanded: 10 20 30 40 50 60  →  Shrunk: 10 20
Program C3
Proper Memory Deallocation using free()

Objective

Demonstrate correct usage of free() — releasing heap memory, setting the pointer to NULL afterwards, and understanding what happens with multiple frees and NULL frees.

free_demo.c#include <stdio.h>
#include <stdlib.h>

int main() {
    /* --- Example 1: Basic free --- */
    int *p = (int *)malloc(5 * sizeof(int));
    if (!p) { printf("Alloc failed\n"); return 1; }

    p[0] = 100;
    printf("Before free: p[0] = %d\n", p[0]);

    free(p);       // memory returned to OS
    p = NULL;      // ← GOOD PRACTICE: prevents dangling pointer

    /* --- Example 2: free(NULL) is always safe --- */
    int *q = NULL;
    free(q);       // no crash — freeing NULL is a no-op
    printf("free(NULL) executed safely\n");

    /* --- Example 3: double free (DANGEROUS — do NOT do this) --- */
    int *r = (int *)malloc(sizeof(int));
    free(r);
    r = NULL;     // set NULL → second free is safe
    free(r);     // safe because r is NULL
    printf("Safe double-free pattern demonstrated\n");

    return 0;
}

What free() Does (and Does NOT Do)

FactDetail
What it releasesThe block of heap memory — it is now available for future allocations.
Does it clear memory?❌ No. The old data may still physically sit there until overwritten.
Does it change the pointer?❌ No. The pointer still holds the old address (now invalid — dangling).
free(NULL)✅ Completely safe — the standard guarantees it is a no-op.
Double free❌ Undefined behaviour — corrupts the heap allocator, may crash or open security holes.
Best practice after freeAlways set ptr = NULL immediately after free(ptr).
Memory Lifecycle Summary malloc/calloc → use the memory → realloc (if size changes) → free → ptr = NULL

8. Static vs Dynamic Allocation

❌ Static (Compile-time)

  • Size must be known before running
  • Wastes memory if too large
  • Crashes if too small (overflow)
  • Lives on the Stack
  • Freed automatically on function return

✅ Dynamic (Run-time)

  • Size decided by user input / file size etc.
  • Allocate exactly what you need
  • Can resize with realloc
  • Lives on the Heap
  • You control when it is freed

9. Common Pitfalls & How to Avoid Them

1. Memory Leak

Allocating memory but never calling free(). The program slowly consumes all RAM.

2. Dangling Pointer

Using a pointer after calling free() on it. Set pointer to NULL after freeing.

3. NULL Dereference

Not checking if malloc returned NULL before using the pointer.

4. Buffer Overflow

Writing past the end of allocated memory — e.g., forgetting the +1 for the null terminator in strings.

5. Double Free

Calling free() twice on the same pointer causes undefined behavior / crash.

6. Wrong Free Order (2-D)

Freeing the parent pointer before the row pointers leaks all row memory. Always free children first.

10. Golden Rules of Dynamic Memory Allocation

Rules to Memorise
  1. Every malloc / calloc / realloc must have a matching free.
  2. Always check if the returned pointer is NULL before using it.
  3. Use sizeof — never hard-code byte counts (e.g., assume int is 4 bytes everywhere).
  4. For strings, allocate length + 1 bytes to hold the null terminator.
  5. For 2-D arrays, free in reverse order: rows first, then the pointer array.
  6. After calling free(ptr), set ptr = NULL to prevent dangling-pointer bugs.
